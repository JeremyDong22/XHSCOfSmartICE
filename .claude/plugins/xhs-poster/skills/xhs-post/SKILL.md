# 小红书餐饮AI自动发帖 Skill

## 使用方式
用户调用 `/xhs-post` 时，根据用户指定的主题和帖子类型，自动完成：
1. 生成帖子内容（封面文字 + 标题 + 正文 + 标签）
2. 通过 Chrome DevTools MCP 在创作服务平台自动发布

## 参数格式
`/xhs-post [主题] [类型:焦虑型|噱头型|干货型|故事型]`
- 示例: `/xhs-post 桌访AI 焦虑型`
- 示例: `/xhs-post 智能库存 干货型`
- 如果用户未指定类型，默认使用"焦虑型"（转化率最高）

## 爆款写作公式

### 标题公式库（从高赞帖子提炼，每次选一种）
1. **情绪冲击+品牌名**: "听完[品牌]AI，我背后发凉" — 用强烈情绪词制造好奇
2. **疑问句式**: "除了XX，AI还将如何改变餐饮业？" — 直接抛出读者心中的疑问
3. **热点工具+行业场景**: "用[热点AI]开餐饮店" — 蹭AI热点嫁接到餐饮
4. **拟人化/比喻**: "给你的餐厅配个AI军师" — 将抽象产品具象化
5. **颠覆认知**: "AI入侵餐饮业：你的店长可能已经过时了" — 用"入侵""颠覆"制造紧迫感
6. **蓝海机会**: "餐饮智能化完全是一片蓝海？！" — 暗示赚钱机会，激发FOMO
7. **身份标签+必看**: "餐饮老板必看！" — 精准圈定目标人群
8. **反差悬念**: "为啥智能化餐饮利润这么高都没人做？" — 用反常识制造悬念

### 开头钩子（前2-3句话必须完成其中一个任务）
- **权威引用**: 引用行业大佬/创始人金句，建立"内部信息"感
- **个人经历**: 用真实身份和经历建立信任，"这个人跟我一样"的共鸣
- **行业观察**: 以观察者视角切入，展示专业度
- **新闻事件**: 借助热点或新产品发布，制造时效性
- **核心规律**: 前2句必须做到→制造情绪波动 / 建立身份认同 / 抛出核心悬念 / 展示独家信息

### 情绪触发词库（写正文时穿插使用）
- **焦虑紧迫**: 背后发凉、军备竞赛、入侵、颠覆、生存题、蓝海、风口、时代红利、被取代
- **震撼惊叹**: 全程高能、重磅、太强了、居然、你猜怎么着
- **行动激励**: 必看、赶紧、拉开差距、跑在前面、占优势
- **痛点共鸣**: 试错成本高、人力有极限、睁一只眼闭一只眼、手忙脚乱、凭经验来
- **价值利益**: 免费、降低成本、提升效率、赚钱、利润
- **口语亲切**: 好吗、谁懂、不是吧、说实话、你猜

### CTA策略（结尾必选一种）
1. **开放式提问**: "不知道你是否也遇到过这种情况？" — 降低评论门槛
2. **关键词评论**: "评论'方案'我私发你" — 精准获取意向客户
3. **免费诊断**: "私信我，免费帮你诊断一次" — 降低咨询门槛
4. **经验交流**: "做餐饮的朋友，你们店是怎么解决的？" — 引导同行讨论
5. **@官方账号**: 末尾@科技薯等官方账号争取流量扶持

### 四种帖子类型

**类型一：焦虑型（转化率最高）**
- 结构: 大品牌数据冲击 → 反问你的店 → 真实客户故事 → 揭露隐藏问题 → AI方案 → CTA
- 封面: 用"提问"模板，大字直击痛点
- 内容模式: 故事+金句+总结（参考703赞的茶百道帖子）

**类型二：噱头型**
- 结构: 震惊标题 → 场景还原 → 反转揭秘 → 产品展示 → CTA
- 封面: 用"涂写"或"边框"模板，蓝色系，制造好奇
- 内容模式: 痛点+解决方案+案例

**类型三：干货型**
- 结构: 痛点共鸣 → 方法论 → 具体步骤(3个) → 效果数据 → CTA
- 封面: 用"备忘"或"基础"模板，白色/黄色系，干净专业
- 内容模式: 列表式知识梳理

**类型四：故事型**
- 结构: 个人经历开头 → 遇到问题 → 解决过程 → 成果 → 感悟 → CTA
- 封面: 用"手写"或"便签"模板，温馨个人化
- 内容模式: 个人实践+工具测评

## 内容生成规则

### 封面文字
- 最多3行，每行不超过8个字
- 用"文字配图"功能生成，选择合适模板
- 焦虑型用"提问"模板，噱头型用"涂写"，干货型用"备忘"，故事型用"手写"

### 标题
- 20字以内，必须有情绪钩子
- 包含目标人群关键词（餐饮老板、店长、餐饮人）

### 正文
- 控制在490-600字（留空间给标签，总上限1000字）
- 段落之间空一行，增加可读性
- 引用客诉/数据时用引号""突出
- 结尾必须有明确CTA

### 标签策略
优先使用系统推荐标签（点击添加），覆盖"行业+技术+热点+场景"四个维度：
- 行业标签（必选）：#餐饮人 #餐饮老板 #餐饮服务业 #餐饮管理
- 技术标签（必选）：#AI #ai工具 #AI人工智能 #餐饮AI
- 热点品牌标签（蹭流量）：#茶百道 #美团 #DeepSeek #海底捞
- 场景标签：#创业 #开一家小店 #餐饮小白 #小成本创业
- 趋势标签：#餐饮未来趋势 #新餐饮模式 #行业发展前景
- 技巧：同时用中英文标签（#AI 和 #ai工具 都用）覆盖更多搜索流量
- 数量：5-10个标签，不要太少

## DevTools MCP 自动发布流程

**前提**: Chrome DevTools MCP 已连接，用户已登录小红书创作服务平台。

### Step 1: 导航到发布页
```
navigate_page → url: "https://creator.xiaohongshu.com/publish/publish?source=official"
```

### Step 2: 检查登录状态
```
take_snapshot → 检查是否有"发布笔记"文字
如果出现登录/二维码页面 → 提示用户手动登录，等待确认
```

### Step 3: 点击"文字配图"
```
take_snapshot → 找到"文字配图"按钮 → click
```

### Step 4: 输入封面文字
```
take_snapshot → 找到文本输入框 → fill(封面文字)
```

### Step 5: 生成图片并选模板
```
click "生成图片" → take_snapshot → 根据帖子类型选择模板:
- 焦虑型 → "提问"
- 噱头型 → "涂写"
- 干货型 → "备忘"
- 故事型 → "手写"
click 对应模板
```

### Step 6: 进入编辑页
```
click "下一步"
```

### Step 7: 填写标题
```
take_snapshot → 找到标题输入框(placeholder="填写标题会有更多赞哦") → fill(标题)
```

### Step 8: 填写正文
```
找到正文输入框(multiline) → click → 全选(Meta+A) → fill(正文内容)
```

### Step 9: 添加标签
```
take_snapshot → 找到系统推荐的标签 → 逐个click添加
每次click后重新take_snapshot获取新推荐标签
添加5-8个相关标签
```

### Step 10: 发布
```
take_snapshot → 找到"发布"按钮 → click
等待"发布成功"确认页面
```

### 异常处理
- 如果页面有草稿箱弹窗遮挡 → navigate_page reload 清除
- 如果"发布"打开新tab → list_pages + select_page 切换
- 如果按钮不可点击 → 等待2秒后重试
- 如果登录过期 → 提示用户手动登录
